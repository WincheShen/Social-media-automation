# 社交媒体自动发布系统 — 需求文档

> 版本：v0.1.0 | 更新日期：2026-03-25

---

## 1. 项目概述

构建一个基于 LangGraph 的多人格社交媒体自动化 Agent 系统，支持：

- **多账号管理**：每个账号拥有独立的人格设定、模型配置和浏览器环境
- **智能内容生成**：根据账号赛道自动检索素材、生成文案和视觉内容
- **自动化发布**：通过浏览器自动化完成社交平台（小红书等）的内容发布
- **数据驱动优化**：基于发布效果数据，持续调优内容策略

---

## 2. 核心功能需求

### 2.1 多人格路由系统 (Identity Router)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-101 | 支持通过 `account_id` 切换完整的人格配置（Persona、模型、风格） | P0 |
| FR-102 | Identity Registry 支持 YAML/JSON 配置，热加载无需重启 | P0 |
| FR-103 | 每个账号独立的 memory 存储，记录历史内容与运营经验 | P0 |
| FR-104 | 支持动态新增/禁用账号，不影响其他账号运行 | P1 |

**Identity Registry 字段定义：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `account_id` | string | ✅ | 唯一标识，如 `XHS_01` |
| `persona` | string | ✅ | 核心人格描述 |
| `track` | string | ✅ | 关注赛道/领域 |
| `platform` | string | ✅ | 目标平台（`xiaohongshu`, `douyin`, `weibo`） |
| `primary_model` | string | ✅ | 主 VLM 模型 |
| `fallback_model` | string | ✅ | 降级模型 |
| `visual_style` | object | ✅ | 视觉风格定义（色调、字体、排版） |
| `proxy_config` | object | ✅ | 代理 IP 配置 |
| `post_schedule` | list | ✅ | 发布时间窗口 |
| `max_daily_posts` | int | ✅ | 每日发布上限 |
| `review_mode` | enum | ✅ | `auto` / `review` / `scheduled` |
| `sensitive_words` | list | ❌ | 赛道特定敏感词库 |
| `system_prompt` | string | ✅ | 该人格的系统提示词 |

### 2.2 多模型研究引擎 (Multi-VLM Research)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-201 | 支持根据任务类型动态选择 VLM（Gemini Pro / Flash / Claude 等） | P0 |
| FR-202 | 支持 Web 搜索、PDF 文档解析、API 数据源（如 yfinance） | P0 |
| FR-203 | 模型调用失败时自动降级到 fallback_model | P0 |
| FR-204 | Token 消耗追踪与成本统计 | P1 |
| FR-205 | 研究结果缓存，避免重复调用（TTL 可配置） | P2 |

**模型适配策略：**

| 任务类型 | 推荐模型 | 理由 |
|---------|---------|------|
| 长文档/政策解读 | Gemini 1.5 Pro | 长上下文窗口，逻辑强 |
| 快速热点扫描 | Gemini 1.5 Flash | 速度快，成本低 |
| 复杂推理/分析 | Claude 3.7 Sonnet | 强逻辑链 |
| HTML 清洗/数据提取 | Gemini 1.5 Flash | 性价比高 |
| 图像理解/审核 | Gemini 1.5 Pro | 多模态能力强 |

### 2.3 内容生成引擎 (Creative Engine)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-301 | 根据 Persona 设定生成对应口吻的文案（标题 + 正文 + 标签） | P0 |
| FR-302 | 支持多种视觉内容生成：思维导图、数据图表、AI 配图 | P0 |
| FR-303 | 文案长度、格式自动适配目标平台规范 | P1 |
| FR-304 | 内容去重检测（基于 embedding 相似度），防止重复发布 | P1 |
| FR-305 | 支持内容模板系统，快速生成系列化内容 | P2 |

**视觉内容生成矩阵：**

| 账号类型 | 视觉需求 | 生成方式 |
|---------|---------|---------|
| 教育类 (XHS_01) | 思维导图、对比表格、知识卡片 | 代码生成 (matplotlib/graphviz) |
| 金融类 (XHS_02) | K 线图、行情标注、数据看板 | API + matplotlib |
| 生活类 (XHS_03) | 温馨配图、大字卡片 | AI 图像生成 (Nano Banana 2) |

### 2.4 内容安全与合规 (Content Safety)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-401 | 平台敏感词过滤（通用 + 赛道专用词库） | P0 |
| FR-402 | 金融类内容自动添加免责声明 | P0 |
| FR-403 | 事实性声明标注（区分"分析观点"与"客观事实"） | P1 |
| FR-404 | 图片合规检测（无水印侵权、无违规内容） | P1 |

### 2.5 审核与发布网关 (Review & Publish Gate)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-501 | 支持三种审核模式：`auto`（自动发布）、`review`（人工审批）、`scheduled`（定时队列） | P0 |
| FR-502 | `review` 模式下，终端/Web UI 展示预览，等待人工确认 | P0 |
| FR-503 | `scheduled` 模式下，内容进入发布队列，按最优时间窗口自动发布 | P1 |
| FR-504 | 金融类账号强制 `review` 模式，不可切换为 `auto` | P0 |

### 2.6 浏览器自动化执行 (Browser Automation)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-601 | 基于 browser-use 实现小红书自动发帖 | P0 |
| FR-602 | 每个账号独立的 Chrome User Profile | P0 |
| FR-603 | 代理 IP 绑定，每账号独立出口 IP | P0 |
| FR-604 | 浏览器指纹差异化（UA、Canvas、WebGL、分辨率） | P1 |
| FR-605 | 操作行为模拟（随机间隔、滚动、点击偏移） | P1 |
| FR-606 | 登录态检测与自动刷新 | P1 |

### 2.7 数据回收与反馈闭环 (Feedback Loop)

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| FR-701 | 发布后 2h / 24h / 72h 自动抓取笔记数据 | P1 |
| FR-702 | 记录维度：曝光、点赞、收藏、评论、分享、涨粉 | P1 |
| FR-703 | 自动生成内容效果分析报告 | P2 |
| FR-704 | 将优化建议写入账号 memory，供下次生成参考 | P2 |
| FR-705 | A/B 测试支持：同话题不同标题/封面对比 | P3 |

## 3. 非功能性需求

### 3.1 可靠性

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| NFR-01 | 任何节点失败不影响其他账号的任务执行 | P0 |
| NFR-02 | API 调用失败自动重试（指数退避，最多 3 次） | P0 |
| NFR-03 | 浏览器崩溃自动恢复，不丢失任务状态 | P1 |
| NFR-04 | LangGraph 状态持久化（Checkpointer），支持断点续跑 | P0 |

### 3.2 安全性

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| NFR-05 | API Key 通过 `.env` 管理，不入版本库 | P0 |
| NFR-06 | 浏览器 Profile 数据加密存储 | P2 |
| NFR-07 | 操作日志脱敏（不记录完整 Cookie / Token） | P1 |

### 3.3 可观测性

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| NFR-08 | 结构化日志（JSON 格式），按账号/节点分类 | P0 |
| NFR-09 | 关键事件告警（账号异常、发布失败）→ 微信/Telegram 通知 | P1 |
| NFR-10 | Token 消耗与 API 成本 Dashboard | P2 |
| NFR-11 | 节点执行耗时追踪 | P1 |

### 3.4 可扩展性

| 需求编号 | 描述 | 优先级 |
|---------|------|--------|
| NFR-12 | 新增平台（抖音、微博）仅需实现 Publisher 接口 | P1 |
| NFR-13 | 新增内容类型仅需注册 CreativeHandler | P1 |
| NFR-14 | 支持多机部署（远程浏览器实例） | P3 |

## 4. 技术约束

| 约束 | 说明 |
|------|------|
| 语言 | Python 3.11+ |
| 编排框架 | LangGraph |
| 浏览器自动化 | browser-use (Playwright-based) |
| LLM Provider | Google Gemini, Anthropic Claude |
| 图像生成 | Nano Banana 2 (远期) |
| 数据存储 | SQLite (本地) → PostgreSQL (扩展) |
| 配置格式 | YAML |
| 包管理 | uv / pip |

## 5. 里程碑规划

| 阶段 | 目标 | 预计周期 |
|------|------|---------|
| **M1 - MVP** | 单账号（XHS_01）完整流程跑通：加载身份 → 生成内容 → 人工审核 → 自动发帖 | 2 周 |
| **M2 - 多账号** | 多账号路由、Profile 隔离、代理 IP 绑定 | 1 周 |
| **M3 - 安全合规** | 敏感词过滤、金融免责声明、内容去重 | 1 周 |
| **M4 - 反馈闭环** | 数据回收、效果分析、Memory 自动更新 | 2 周 |
| **M5 - 高级功能** | 定时发布队列、A/B 测试、多平台扩展 | 持续迭代 |
